自检 12：响应式方案
请说明你知道的：
● 媒体查询
● 流式布局
● rem / vw 方案
并回答：
移动端你更倾向用哪种？为什么？

1. 媒体查询
媒体查询是根据设备的屏幕宽度、分辨率等条件，给不同尺寸的屏幕应用不同的样式。
它的核心作用是：在不同终端下做样式适配。例如同一套页面，在手机、平板、PC 上显示不同布局。
常见写法：
@media (max-width: 768px) {
  .box {
    width: 100%;
  }
}
适合场景：PC 和移动端布局差异比较大,需要做断点适配

2. 流式布局
流式布局是指元素宽度不写死，而是使用百分比等相对单位，让布局随着容器宽度变化而自动伸缩。
比如：
.container {
  width: 100%;
}
.left {
  width: 30%;
}
.right {
  width: 70%;
}
它的特点是：布局更灵活,能随屏幕宽度自动变化,但如果屏幕过大或过小，可能会出现拉伸过度或内容拥挤的问题

3. rem / vw 方案
rem 方案:rem 是相对于根元素 html 的字体大小来计算的。
移动端常见做法是动态设置 html 的 font-size，让整个页面按照屏幕宽度等比缩放。
它的特点：适合做移动端整体等比适配,页面元素可以按设计稿比例转换,兼容性较好，应用比较广
vw 方案:vw 是视口宽度的百分比单位，1vw = 视口宽度的 1%。
直接根据屏幕宽度计算尺寸，不依赖根字体大小。
它的特点：写法更直接,适合做响应式宽度适配,在移动端也比较常用

4. 移动端更倾向用哪种？为什么？
如果是纯移动端项目，我一般更倾向用 vw 或 rem 方案，具体要看项目要求。
我自己的倾向：
如果是等比适配、设计稿转换比较多，我会更倾向 rem
如果是新项目、布局想更直接，我会更倾向 vw
原因：
适配效果更稳定,能随着屏幕宽度变化自动缩放,开发效率高
不需要为很多屏幕单独写样式,更符合移动端多尺寸场景,适配不同手机屏幕更方便
5. 面试回答可以这样说
我知道常见的响应式方案有媒体查询、流式布局和 rem/vw。
媒体查询主要是根据屏幕断点切换不同样式，适合 PC 和移动端差异比较大的场景。
流式布局是用百分比等相对单位，让元素随容器宽度变化自动伸缩。
rem 和 vw 更常用于移动端适配，rem 是基于根字体大小缩放，vw 是基于视口宽度计算。
如果是移动端项目，我通常会优先考虑 vw 或 rem，因为它们更适合多尺寸屏幕的等比适配，开发成本也更低。
