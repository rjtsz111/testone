自检 13：隐藏元素的方式
对比下面方案的区别：
● display: none
● visibility: hidden
● opacity: 0
关注点：
● 是否占位
● 是否响应事件
● 是否影响性能 / 重绘

1. display: none
特点:元素不显示,不占位,从渲染树中移除，页面布局会重新计算
是否响应事件:不响应,因为元素根本不参与渲染和交互
是否影响性能 / 重绘:会触发回流（reflow）
因为元素的占位消失，布局需要重新计算,性能开销相对更大
适用场景:完全不需要显示时,需要从布局中移除元素时

2. visibility: hidden
特点:元素不可见,占位,只是视觉上隐藏，布局空间还在
是否响应事件:通常不响应鼠标点击等交互,因为元素虽然占位，但不可见，浏览器不会正常让它参与用户交互
是否影响性能 / 重绘:一般只会引起重绘（repaint）,不会像 display: none 那样引起布局结构变化,性能开销比 display: none 小一些
适用场景:需要保留布局位置，但暂时隐藏内容,比如表格行、占位元素等

3. opacity: 0
特点:元素完全透明，占位，视觉上看不见，但元素仍然存在于页面中
是否响应事件：会响应事件，这是和 visibility: hidden 最大的区别之一，虽然看不见，但它依然能被点击、hover、聚焦等
是否影响性能 / 重绘：通常会触发重绘，不会导致布局变化，动画过渡时常用，因为可以做渐隐效果
适用场景：需要做淡入淡出动画，需要隐藏但保留交互能力的场景
对比表
属性	              是否占位	是否响应事件	主要影响
display: none	        否	     否	       回流 + 重绘
visibility: hidden	  是	     否	        重绘
opacity: 0	          是	     是	        重绘

面试回答可以这样说
display: none 是彻底从布局中移除，不占位，也不响应事件，通常会引起回流。
visibility: hidden 是隐藏但保留占位，元素不显示，一般不响应事件，主要影响重绘。
opacity: 0 是透明隐藏，元素仍然占位，而且仍然可以响应事件，常用于动画过渡场景。
所以三者区别主要在于是否占位、是否可交互，以及对布局和渲染性能的影响。

一句话总结
display: none 是“消失并移除”，visibility: hidden 是“看不见但还在”，opacity: 0 是“透明但仍可交互”。
